# Use Cases — Clive `--explore` pipeline (Given/When/Then)

Derived from the happy_path, error_path, and state_transition scenarios in scenarios.md. Each use case is testable end-to-end.

## UC-1: Successful exploration of a read-only batch tool

- **Given** `rg` is installed, no `drivers/rg.md` exists, the LLM provider is reachable, and tmux is running,
- **When** the user runs `python clive.py --explore rg`,
- **Then** the system creates an isolated tmux session + `/tmp/clive/explore-rg-<hex>/`, runs 5–8 read-only probes (`--help`, `--version`, `man rg | head -80`, two example invocations) ending with a `DONE:` summary,
- **And** the synthesizer returns valid driver markdown with frontmatter + ENVIRONMENT/PRIMARY TOOLS/PATTERNS/RESPONSE FORMAT/COMPLETION,
- **And** `drivers/rg.md` is written with the auto-gen header inside the body,
- **And** the CLI prints `Wrote driver: drivers/rg.md` + the summary,
- **And** the process exits 0.

## UC-2: Refuse to overwrite an existing driver

- **Given** `drivers/git.md` already exists (hand-written),
- **When** the user runs `clive --explore git` (no overwrite flag),
- **Then** exploration completes,
- **And** `write_generated_driver` raises `FileExistsError`,
- **And** the CLI prints `driver already exists at drivers/git.md; pass overwrite=True to replace` + `Re-run with --explore-overwrite to replace.`,
- **And** the existing driver content is preserved byte-for-byte,
- **And** the process exits 2.

## UC-3: Refuse path-traversal in tool name

- **Given** any working environment,
- **When** the user runs `clive --explore "../../etc/passwd"`,
- **Then** `_SAFE_NAME` rejects the name,
- **And** `write_generated_driver` raises `ValueError("unsafe tool name for driver path: '../../etc/passwd'")`,
- **And** no file is written outside `drivers/`,
- **And** the process exits non-zero.

> **GAP:** today this check fires only at write time. UC-3 should also imply that no exploration pane is created and no LLM tokens are spent (see scenarios.md #5, #16).

## UC-4: Report a synthesizer that returns an incomplete driver

- **Given** exploration completes with N probes (N > 0),
- **And** the synthesizer LLM returns markdown missing the `PATTERNS` section,
- **When** `generate_driver` validates the response,
- **Then** it raises `ValueError("driver for <tool> missing section(s): PATTERNS")`,
- **And** the CLI prints `Driver synthesis failed: ...`,
- **And** no file is written,
- **And** the process exits 1.

> **GAP:** the exploration result is discarded — no caching, no retry path. See scenario #2.

## UC-5: Refuse a CREDENTIAL_TOOLS / INTERACTIVE_TOOLS invocation without help flag

- **Given** an exploration pane is open,
- **When** the LLM emits ```bash aws s3 ls``` to the pane,
- **Then** `_check_exploration_safety` returns `Blocked: 'aws' may prompt for credentials. Append --help or --version, or skip this probe.`,
- **And** the command is not executed,
- **And** the runner feeds the violation back to the LLM as a screen update.

## UC-6: Allow CREDENTIAL_TOOLS with explicit help flag

- **Given** an exploration pane is open,
- **When** the LLM emits ```bash aws --help```,
- **Then** `_check_exploration_safety` returns `None`,
- **And** the command runs in the pane.

## UC-7: Apply environment-variable stripping before the credential guard

- **Given** an exploration pane is open,
- **When** the LLM emits ```bash AWS_PROFILE=foo aws --help```,
- **Then** `_check_exploration_safety` strips `AWS_PROFILE=foo`,
- **And** evaluates `aws --help`,
- **And** returns `None` (passes).

## UC-8: Overwrite an existing driver only with the explicit flag

- **Given** `drivers/rg.md` exists,
- **When** the user runs `clive --explore rg --explore-overwrite`,
- **Then** the existing file is truncated and replaced,
- **And** the process exits 0.

> **GAP (UC-8 risk, scenario #12 + #33):** today this same flag overwrites hand-written drivers identically to auto-gen. Recommended split: `--refresh` (auto-gen only) vs `--force-overwrite` (any).

## UC-9: Auto-gen header is preserved inside the body, not before frontmatter

- **Given** a valid synthesized driver text starting with `---\n...frontmatter...\n---\n# tool Driver\n...`,
- **When** `_inject_header` runs,
- **Then** the returned text still starts with `---`,
- **And** the substring `Auto-generated by clive --explore` appears in the first 200 bytes after the closing `---`,
- **And** `_parse_driver_frontmatter` (in `llm/prompts.py`) can still extract the frontmatter unchanged.

## UC-10: ExplorationResult aggregates probes

- **Given** `explore_tool("foo")` runs in a mocked runner that emits 3 `probe` events,
- **When** the runner returns,
- **Then** `result.tool_name == "foo"`,
- **And** `len(result.probes) == 3`,
- **And** `result.success_count + result.failure_count == 3`.
